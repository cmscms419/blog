# VKShaderManager와 VKshader 클래스 구조 분석

## 개요

VKShaderManager와 VKshader 클래스는 Vulkan 기반 렌더링 엔진에서 SPIR-V 쉐이더 파일 관리와 파이프라인별 쉐이더 조직화를 담당하는 핵심 컴포넌트입니다. 
이 시스템은 SPIR-V 리플렉션을 활용하여 런타임에 쉐이더 메타데이터를 자동으로 추출하고, 디스크립터 레이아웃을 동적으로 생성합니다.

## 클래스 구조
### 1. VKshader 클래스
```cpp
class VKshader {
    friend class VKShaderManager;
    
private:
    VkContext& ctx;                           // Vulkan 컨텍스트 참조
    VkShaderModule module;                    // Vulkan 쉐이더 모듈
    SpvReflectShaderModule reflectModule;     // SPIR-V 리플렉션 모듈
    VkShaderStageFlagBits stage;             // 쉐이더 스테이지 (Vertex, Fragment 등)
    cString name;                            // 쉐이더 파일명
};
```
주요 특징:
•	Move-Only 클래스: 복사 생성자와 할당 연산자가 삭제됨
•	RAII 패턴: 소멸자에서 자동으로 Vulkan 리소스 정리
•	Friend 클래스: VKShaderManager에서만 private 멤버 접근 가능
### 2. VKShaderManager 클래스

```cpp
class VKShaderManager {
private:
    VkContext& ctx;                                               // Vulkan 컨텍스트
    std::unordered_map<cString, std::vector<VKshader>> pipelineShaders;  // 파이프라인별 쉐이더 그룹
    std::vector<LayoutInfo> layoutInfos;                         // 디스크립터 레이아웃 정보
};

핵심 데이터 구조:
•	pipelineShaders: 파이프라인 이름을 키로 하는 쉐이더 벡터 맵
•	layoutInfos: 통합된 디스크립터 레이아웃 정보 배열
```
## 데이터 흐름 및 초기화 과정

### 1. VKshader 생성 과정
```
[.spv 파일] → [바이너리 읽기] → [Vulkan 모듈 생성] → [리플렉션 모듈 생성] → [메타데이터 추출]
     ↓              ↓                 ↓                   ↓                 ↓
  filepath  helper::readSPVFile  VkShaderModule  SpvReflectShaderModule stage, bindings
  
초기화 단계:
1.	파일 읽기: helper::readSPVFile()로 SPIR-V 바이너리 로드
2.	모듈 생성: helper::createShaderModule()로 VkShaderModule 생성
3.	리플렉션: helper::createSpvReflectModule()로 메타데이터 추출
4.	스테이지 결정: 리플렉션 데이터에서 쉐이더 타입 자동 감지
```

### 2. VKShaderManager 초기화
```cpp

// 사용 예시
VKShaderManager shaderManager(ctx, "shaders/", {
    {"basic_pipeline", {"basic.vert", "basic.frag"}},
    {"pbr_pipeline", {"pbr.vert", "pbr.frag"}},
    {"compute_pipeline", {"particle.comp"}}
});

[초기화 리스트] → [createFromShaders()] → [각 쉐이더 로드] → [collectLayoutInfos()] → [레이아웃 통합]

```

## 리플렉션 데이터 구조
-> 쉐이더에 정보를 바탕으로 어떤 정보가 필요한지 들어있음
### SpvReflectShaderModule 주요 멤버
```cpp
typedef struct SpvReflectShaderModule {
    const char* entry_point_name;                    // "main" 등 엔트리 포인트
    SpvReflectShaderStageFlagBits shader_stage;      // 쉐이더 스테이지
    
    // 디스크립터 바인딩 정보
    uint32_t descriptor_binding_count;               // 바인딩 개수
    SpvReflectDescriptorBinding* descriptor_bindings; // 바인딩 배열
    
    // 입력 변수 정보 (버텍스 쉐이더용)
    uint32_t input_variable_count;                   // 입력 변수 개수
    SpvReflectInterfaceVariable** input_variables;   // 입력 변수 배열
    
    // 푸시 상수 정보
    uint32_t push_constant_block_count;              // 푸시 상수 블록 개수
    SpvReflectBlockVariable* push_constant_blocks;   // 푸시 상수 블록 배열
} SpvReflectShaderModule;
```
### 디스크립터 바인딩 정보
```cpp
typedef struct SpvReflectDescriptorBinding {
    const char* name;                    // 바인딩 변수명
    uint32_t binding;                    // 바인딩 인덱스
    uint32_t set;                        // 디스크립터 셋 인덱스
    SpvReflectDescriptorType descriptor_type;  // 바인딩 타입 (UBO, Texture 등)
    uint32_t count;                      // 배열 크기
} SpvReflectDescriptorBinding;
```
## 레이아웃 정보 수집 과정
### 1. collectPerPipelineBindings() 메서드

**파이프라인의 모든 쉐이더에서 디스크립터 바인딩 정보 수집**이 목적이다

```cpp
// 데이터 구조: setIndex → bindingIndex → VkDescriptorSetLayoutBinding
std::map<uint32_t, std::map<uint32_t, VkDescriptorSetLayoutBinding>> pipelineBindingCollector;
```
[파이프라인 쉐이더들] → [각 쉐이더 리플렉션] → [바인딩 정보 추출] → [중복 검사] → [스테이지 플래그 누적]

핵심 로직:
•	동일한 바인딩이 여러 쉐이더에서 사용되면 stageFlags 누적
•	try_emplace()를 사용한 효율적인 중복 처리
### 2. collectLayoutInfos() 메서드
목적: 전체 파이프라인에서 중복되는 레이아웃 제거 및 통합
```cpp
// 중복 제거를 위한 컨테이너
std::unordered_map<
    std::vector<VkDescriptorSetLayoutBinding>,  // Key: 정규화된 바인딩
    std::vector<std::tuple<cString, uint32_t>>, // Value: 파이프라인명+셋번호
    BindingHash,
    BindingEqual
> bindingCollector;
```
```
bindingCollector
├── Key: vector<VkDescriptorSetLayoutBinding> [정규화됨]
│   ├── [0] VkDescriptorSetLayoutBinding {binding=0, type=UNIFORM_BUFFER, ...}
│   ├── [1] VkDescriptorSetLayoutBinding {binding=1, type=COMBINED_IMAGE_SAMPLER, ...}
│   └── ...
│
└── Value: vector<tuple<cString, cUint32_t>>
    ├── [0] tuple<"pipeline1", 0>  // 파이프라인명, setIndex
    ├── [1] tuple<"pipeline2", 0>
    └── ...
1.	레이아웃 중복 제거: 동일한 바인딩 구조를 가진 레이아웃들을 하나로 통합
2.	정규화: stageFlags를 0으로 설정하여 비교 기준 통일
3.	효율적 검색: 해시맵 사용으로 O(1) 평균 검색 시간
4.	메타데이터 보존: 각 레이아웃을 사용하는 파이프라인들의 정보 유지

```

정규화 과정:
1.	모든 바인딩의 stageFlags를 0으로 초기화
2.	실제 사용되는 스테이지 플래그 누적
3.	동일한 바인딩 구조를 하나의 키로 통합
-> 예시): pipeline1, pipeline2의 모든 바인딩 플러그를 동일하게 만든다.
•	Pipeline A가 VK_SHADER_STAGE_VERTEX_BIT만 사용
•	Pipeline B가 VK_SHADER_STAGE_FRAGMENT_BIT만 사용
•	같은 바인딩 구조를 가진다면
→ 최종적으로 둘 다 VK_SHADER_STAGE_VERTEX_BIT | VK_SHADER_STAGE_FRAGMENT_BIT로 통일됩니다.
## 파이프라인 생성 지원 기능

### 1. 쉐이더 스테이지 정보 생성
```cpp
std::vector<VkPipelineShaderStageCreateInfo> createPipelineShaderStageCIs(cString pipelineName);
```
생성되는 정보:
•	stage: 쉐이더 스테이지 (VK_SHADER_STAGE_VERTEX_BIT 등)
•	module: VkShaderModule 핸들
•	pName: 엔트리 포인트 이름 (리플렉션에서 자동 추출)
### 2. 버텍스 입력 속성 생성
```cpp
std::vector<VkVertexInputAttributeDescription> createVertexInputAttrDesc(cString pipelineName);
```
처리 과정:
1.	파이프라인에서 버텍스 쉐이더 찾기
2.	리플렉션 데이터에서 입력 변수 추출
3.	VkVertexInputAttributeDescription 배열 생성
### 3. 푸시 상수 범위 생성
```cpp
VkPushConstantRange pushConstantsRange(cString pipelineName);
```
기능:
•	파이프라인의 모든 쉐이더에서 푸시 상수 검출
•	여러 쉐이더가 푸시 상수를 사용하면 stageFlags 누적
•	크기와 오프셋 정보 제공
## LayoutInfo 구조체
```cpp
struct LayoutInfo {
    std::vector<VkDescriptorSetLayoutBinding> bindings;           // 바인딩 정보
    std::vector<std::tuple<cString, uint32_t>> pipelineNamesAndSetNumbers;  // 사용하는 파이프라인들
};
```
역할:
•	동일한 디스크립터 레이아웃을 사용하는 파이프라인들을 그룹화
•	메모리 효율성을 위한 레이아웃 공유
•	DescriptorManager2와 연동하여 디스크립터 셋 생성
## 데이터 변환 과정
### SPIR-V → Vulkan 바인딩 변환
```cpp
VkDescriptorSetLayoutBinding createLayoutBindingFromReflect(
    const SpvReflectDescriptorBinding* binding,
    VkShaderStageFlagBits shaderStage
);
```
변환 규칙:
•	binding→binding: 바인딩 인덱스 직접 매핑
•	binding→descriptor_type: 디스크립터 타입 변환
•	SAMPLED_IMAGE → COMBINED_IMAGE_SAMPLER 자동 변환
•	binding→count: 배열 크기 정보
•	shaderStage: 해당 쉐이더 스테이지 플래그
## 메모리 관리 전략
### 1. Move Semantics 활용
```cpp
VKshader(VKshader&& other) noexcept;           // 이동 생성자만 허용
VKshader(const VKshader&) = delete;            // 복사 생성자 삭제
```
### 2. RAII 패턴
```cpp
VKshader::~VKshader() {
    cleanup();  // 자동 리소스 정리
}

void VKshader::cleanup() {
    if (reflectModule._internal != nullptr) {
        spvReflectDestroyShaderModule(&reflectModule);
    }
    if (module != VK_NULL_HANDLE) {
        vkDestroyShaderModule(ctx.getDevice()->logicaldevice, module, nullptr);
    }
}

```
### 3. 효율적인 컨테이너 사용
•	std::unordered_map: 빠른 파이프라인 검색
•	std::vector::reserve(): 메모리 재할당 최소화
•	emplace_back(): 불필요한 복사 방지
