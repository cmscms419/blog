3D object를 연구한다

- Scalas : 실제 값
- Vectors : 방향 + 크기
- Point : 공간의 위치
- 연산 규칙
    - vector = point - point
    - point = point + point
    - vector = vector + vector

## 벡터 공간

스칼라와 벡터가 정의되어 있는 공간이다.

- 스칼라-벡터 곱 ($v = 2u$)
- 벡터-벡터 더하기 ($w = u +v$)
- 벡터 = 단위 벡터의 합 ($w = αi +βj +γk$)

## 어파인 공간

벡터 공간에 points가 추가 되었다.

- vector-point addition (p = q + v)
- point-point subtraction(v = p – q)

## 좌표계와 프레임

basis vectors : 좌표축 상의 유닛 벡터

좌표계 시스템은 basis vector로 정의 되어 있다.

frame : 참조점 추가

- 보통 좌표계와 혼용
- basis vector: i,j,k 또는 $v1, v2, v3$
- 참조점은 o로 표기

좌표 = 원점 + basis vecter의 합

- $p = o + 2i + 3j + 4k$

### homogeneous 좌표계

4D point = generalized point

- 4D 공간에서 w로 나누면 → (x/w, y/w, z/w, 1)
- 3D 공간의 한 점의 point가 된 (x/w, y/w, z/w)
- 만약 w=0이면, 3d 공간에서는 vecter가 된다 → (x, y, z)

3D에서의 변환

- 3D 공간에서의 (x, y, z)한 점이 → 4D 공간에서는 (x, y, z, 1)
- 또한, 4D 공간에서의 (wx, wy, wz, w)으로 정의 할 수 있다.

# back face culling

## Winding Rule

normal vector

- 법선 벡터
- 삼각형에 수직인 벡터, 바깥쪽을 가르킨다.

normal vector

- 오른손 법칙 : CCW 반시계 방향
- 왼손 법칙 : CW 시계 방향

### 카메라 방향 기준

- front face 앞면 : 카메라와 마주본다
- back face 뒤면 : 카메라와 같은 방향

3D 입체라면

- 모든 방향을 면으로 둘러싼다
- back face는 front face가 덮어씀

back face culling(뒷면 추출)

- primitive assembly  시에, back face는 pipeline에서 추출(cull)한다.

back face culling

- 장점
    - 평균적으로 face 개수의 절반이 cull됨
    - 약 속도가 2배 정도 빨라질 수 있다.
- 단점
    - 못쓰는 경우
    - 투명한 물체
    - 완전히 둘러싸이지 않은 물체

### normal vector 뒤집기

- 만약, 사물안에 들어가게 되면, front normal 벡터를 뒤집어야 한다.

→ CCW에서 CW로 전환해야 한다.

### OpenGL function

- void glFrontFace(GLenum mode)
    - 앞면으로 인식할 **법선 벡터**(CW, CCW)를 결정한다.
    - front와 back facing 폴리곤을 정의한다.
    - mode : : GL_CCW “**counter-clockwise”** (default), GL_CW ”**clockwise”**
- void glCullFace(GLenum mode)
    - front, back, front-back 중 어는 법선 벡터의 방향을 지울 것인지 결정한다.
- glEnable(GL_CULL_FACE)
- glDisable(GL_CULL_FACE)
