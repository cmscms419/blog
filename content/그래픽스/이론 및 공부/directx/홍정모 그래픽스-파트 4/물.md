# 물

밀도장에서 연기는 사라진다. 

하지만 물은 사라지면 이상하다. 

입자 시뮬레이션은 상호작용을 해야한다.  → 하지만, 물에서 사용하기에는 불안정하다.

격자 시뮬레이션은 미분하는 방향이 정해져 있어서, 안정적이다.

입자는 계속 움직여서 불안하다. 보안할 수 있지만, 기본 알고리즘은 격자가 더 안정적

격자 안정성 + 입자 상세도 = 물 시뮬레이션

실제로는 시뮬레이션에 따라서 표현되는 방법이 다르다.

연기는 Raycast

물은 명확하게 한다.

입자가 많은 곳 → 밀도가 높다. → 볼룸 랜더링 하는 것처럼 할 수 있다.

density하는 것은 입자가 겹치는 것은 사실성이 좀 낮다.

SDF → Mesh 바꿔서 만들 수 있다.

## 코드

`ParticleStep()`에서는 입자를 업데이트 시켜준다.

속도의 차이를 입자에게 적용 시킨다.

입자 단위로 새로운 파티클 추가 → 새로운 파티클의 속도를 지정하면 → 초기 속도

GPU에서는 sort을 한다. → 픽셀의 중심 위치도 입자처럼 생각할 수 있다. 픽셀 가까이에 있는 입자를 찾기 위해서 sort을 한다.

### Ex1606_Common.hlsli

```cpp
struct SortElement
{
	uint key;
	uint value;
};
```

- key
    - 어떤 입자가 포함되어 있는 shell의 1차원 index이다.
    - 1차원 index들에 따라서 정렬 -> 입자가 간접적으로 위치에 따라서 정렬이 되는 것처럼 될 수 있다.
- value
    - 파티클이 저장되어 있는 index이다.
    - key에 따라서 정렬이 된다.

### Ex1606_DiffUpSampleCS.hlsl

```cpp
SortElement ref = sortElements[dtID.x];

float3 uvw = particles[ref.value].pos.xyz * dxBase;
float d = densityNew.SampleLevel(linearClampSampler, uvw, 0).r;
float s = signedDistance.SampleLevel(linearClampSampler, uvw, 0).r;

if (d > 1e-5 && s < -0.1 && ref.key != INACTIVE) // Active particle
{
    float3 velOld = velocityOld.SampleLevel(linearClampSampler, uvw, 0).xyz;
    float3 velNew = velocityNew.SampleLevel(linearClampSampler, uvw, 0).xyz;
    
    particles[ref.value].vel = lerp(velNew, particles[ref.value].vel + velNew - velOld, coeff);
}
```
