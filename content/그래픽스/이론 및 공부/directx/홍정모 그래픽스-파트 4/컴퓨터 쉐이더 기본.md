# 격자 무늬

픽셀을 그룹 단위로 나누어서 스레드를 나누어서 렌더링한다.

[ID3D11DeviceContext::D ispatch(d3d11.h) - Win32 apps](https://learn.microsoft.com/ko-kr/windows/win32/api/d3d11/nf-d3d11-id3d11devicecontext-dispatch)

여러 쓰레드는 “그룹”단위로 관리한다.

그래서 우리가 **스레드 그룹의 개수**와 **한 그룹에서 사용할 스레드 개수**를 **지정**해주는 것이다.

![Untitled](attachments/Untitled_17.png)

**Dispatch(5,3,2)** : 쓰레드에게 명령할 “쓰레드 그룹”의 숫자를 정해준다.

- x : 5, y : 3, z : 2
- 위 숫자에 해당하는 숫자로 나누어서 쓰레드 그룹을 만듭니다.

**Zoom-in on SV_Group(2,1,0)**: SV_Group(2,1,0) 부분의 그룹을 “확대한” 부분입니다.

**SV_GroupThreadID(7,5,0)** : SV_GroupID(2,1,0)에 해당하는 그룹 안에서의 정해진 쓰레드 index

### CPU

스레드 그룹의 개수를 정해준다.

```cpp
m_context->Dispatch(UINT(ceil(m_screenWidth / 32.0f)),
                    UINT(ceil(m_screenHeight / 32.0f)), 1);
```

### Compute Shader

```cpp
RWTexture2D<float4> gOutput : register(u0);
cbuffer MyBuffer : register(b0)
{
    float scale;
}
// TODO: numthreads 공식 문서 확인

[numthreads(32, 32, 1)]
void main(int3 gID : SV_GroupID, uint3 tID : SV_DispatchThreadID)
{
    //TODO: groupID, DTid 등을 여러가지로 바꿔가면서 테스트

    //if (t.x % uint(2) == 0)
    //if (distance(float2(250, 250), float2(tID.x, tID.y)) <= 50)
    if (gID.x % uint(2) != gID.y % uint(2))
    {
        // gOutput[tID.xy] : 전체 픽셀의 위치 반환
        gOutput[tID.xy] = float4(0.5, 0.5, 0.5, 1.0);
    }
    else
    {
        gOutput[tID.xy] = float4(1, 1, 1, 1) * scale;
    }
}
```

```cpp
RWTexture2D<float4> gOutput : register(u0);
```

- RWTexture2D : read and write가 가능하다.
- float4 : 텍스처2d 포맷이 float4라는 의미이다.
- u0 : 0 : 슬롯 번호가 0번이다.
- u : CSSetUnorderedAccessViews에 들어가는 주소
- 쉐이더 안에서 RWTexture2D<float4> 처럼 사용된다.
