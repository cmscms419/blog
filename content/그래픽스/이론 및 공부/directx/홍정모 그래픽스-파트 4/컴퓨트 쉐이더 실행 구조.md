# 컴퓨트 쉐이더 실행 구조

```cpp
DXGI_SWAP_CHAIN_DESC sd;
 ZeroMemory(&sd, sizeof(sd));
 sd.BufferDesc.Width = m_screenWidth;
 sd.BufferDesc.Height = m_screenHeight;
 sd.BufferDesc.Format = m_backBufferFormat;
 sd.BufferCount = 2;
 sd.BufferDesc.RefreshRate.Numerator = 60;
 sd.BufferDesc.RefreshRate.Denominator = 1;
 sd.BufferUsage = DXGI_USAGE_RENDER_TARGET_OUTPUT |
                  DXGI_USAGE_UNORDERED_ACCESS; // Compute Shader
// 먼저 사용할 수 있게 BufferUsage에 사용할 수 있게 해준다.
```

Compute Shader에서 BackBuffer를 사용하기 위해서는 FLOAT로 설정

```cpp
AppBase::m_backBufferFormat = DXGI_FORMAT_R16G16B16A16_FLOAT;
```

- m_texA : 실제 데이터
- m_rtvA : m_texA를 Rendertarget으로 보는 view
- m_srvA : m_texA를 ShaderResourseView으로 보는 view
- m_uavA : m_texA를 UnorderedAccessViews으로 보는 view

view 3개를 동시에 사용할 수 없다.

어떤 것은 쓰고, 읽고, 보내는 것을 하면 혼란 발생

vertex shader에서 계산한 pixel들의 위치를 pixel shader에서는 그 pixel의 위치를 바꿀 수 없다.

→ pixel shader에서는 원하는 pixel의 색깔 값을 정할 수 없다.

하지만 compute shader에서는 가능하다.

DXGI_USAGE_UNORDERED_ACCESS이 필요하다.

```cpp
m_context->Dispatch(UINT(ceil(m_screenWidth / 256.0f)), m_screenHeight, 1);
// m_context->Dispatch(가로, 세로, 깊이);
```

[ID3D11DeviceContext::D ispatch(d3d11.h) - Win32 apps](https://learn.microsoft.com/ko-kr/windows/win32/api/d3d11/nf-d3d11-id3d11devicecontext-dispatch)

위 참고

Dispatch : 일감을 나눠서 일을 시킨다.

→ 아주 많은 쓰레드들에게 일감을 나눠준다.

`1` : 장면이 1장이다.

만약 3d이면, depth와 관련된 값을 넣으면 된다.

```cpp
ID3D11ShaderResourceView *nullSRV[6] = {
    0,
};
m_context->CSSetShaderResources(0, 6, nullSRV);
ID3D11UnorderedAccessView *nullUAV[6] = {
    0,
};
m_context->CSSetUnorderedAccessViews(0, 6, nullUAV, NULL);
```

compute shader는 여러 스레드로 일을 한다.
전부 끝나야 다음 일을 할 수 있으니, 대기하고 있으라고 명령하는 것이다.
CPU에서 명령을 내리는 것으로, 내부적으로는 GPU에서 일을 끝날때까지 기다린다.

[ID3D11DeviceContext::D ispatch(d3d11.h) - Win32 apps](https://learn.microsoft.com/ko-kr/windows/win32/api/d3d11/nf-d3d11-id3d11devicecontext-dispatch)
