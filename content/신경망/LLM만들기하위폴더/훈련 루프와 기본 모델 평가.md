**핵심 공식**: `new_weight = old_weight - learning_rate × gradient`
### 교차 엔트로피(cross_entropy)

개념 : 교차 엔트로피는 모델의 예측 확률분포와 실제 정답 간의 차이를 측정하는 손실 함수입니다.
-> 신경 네트워크에서 말하는 loss를 계산하는 함수라고 생각하면 된다.

- 아래 그림은 매우 작은 어휘를 예시로 사용하여, 이전 장의 마지막에서 논의한 확률 점수를 텍스트로 다시 변환하는 방법을 보여줍니다
![](https://sebastianraschka.com/images/LLMs-from-scratch-images/ch05_compressed/proba-to-text.webp)
- 이전 장에서 논의했듯이, `argmax` 함수를 적용하여 확률 점수를 예측 토큰 ID로 변환할 수 있습니다
- 위의 소프트맥스 함수는 각 토큰에 대해 50,257 차원 벡터를 생성했고, `argmax` 함수는 이 벡터에서 가장 높은 확률 점수의 위치를 반환하여 해당 토큰의 예측 토큰 ID가 됩니다
- 각각 3개의 토큰을 가진 2개의 입력 배치가 있으므로, 2 x 3 예측 토큰 ID를 얻습니다:
- 실제 정답은 "effort" (인덱스 1)인데, 모델은 0.60의 확률을 부여했습니다
- 이 토큰들을 디코딩하면, 모델이 예측하기를 원하는 타겟 토큰들과는 상당히 다르다는 것을 알 수 있습니다:
- 모델을 훈련하려면 올바른 예측(타겟)으로부터 얼마나 멀리 떨어져 있는지 알아야 합니다
![](https://sebastianraschka.com/images/LLMs-from-scratch-images/ch05_compressed/proba-index.webp)
- 타겟 인덱스에 해당하는 토큰 확률들은 다음과 같습니다:
- 이 모든 값들을 최대화하여 1에 가까운 확률로 만들고 싶습니다
- 수학적 최적화에서는 확률 점수 자체보다 확률 점수의 로그를 최대화하는 것이 더 쉽습니다. 이는 이 책의 범위를 벗어나지만, 더 자세한 내용이 담긴 강의를 여기에 녹화해두었습니다: [L8.2 Logistic Regression Loss Function](https://www.youtube.com/watch?v=GxJe0DZvydM)
- 다음으로, 평균 로그 확률을 계산합니다:
- 목표는 모델 가중치를 최적화하여 이 평균 로그 확률을 가능한 한 크게 만드는 것입니다
![](https://sebastianraschka.com/images/LLMs-from-scratch-images/ch05_compressed/cross-entropy.webp?123)
- 로그 때문에 가능한 가장 큰 값은 0이고, 우리는 현재 0에서 멀리 떨어져 있습니다
- 딥러닝에서는 평균 로그 확률을 최대화하는 대신, *음의* 평균 로그 확률 값을 최소화하는 것이 표준 관례입니다. 우리의 경우, -10.7722를 최대화하여 0에 가깝게 만드는 대신, 딥러닝에서는 10.7722를 최소화하여 0에 가깝게 만듭니다
- -10.7722의 음수 값, 즉 10.7722는 딥러닝에서 교차 엔트로피 손실이라고도 불립니다
- 타겟은 토큰 ID이며, 이는 또한 최대화하려는 로짓 텐서의 인덱스 위치를 나타냅니다
- PyTorch의 `cross_entropy` 함수는 최대화해야 하는 로짓의 토큰 인덱스에 대해 softmax와 로그 확률 계산을 내부적으로 자동으로 처리합니다
- 교차 엔트로피 손실과 관련된 개념은 LLM의 perplexity입니다
- Perplexity는 단순히 교차 엔트로피 손실의 지수입니다
### Perplexity
- 개념 : cross_entropy  지수로, 모델이 얼마나 "당황하고 있는지"를 나타냅니다.
- tensor(48725.8203)의 의미는 각 단계에서 약 48725개의 토큰(단어)중에서 고민하고 있다는 의미이다.
- 값이 낮으면 확신이 높음, 값이 높으면 불확실성이 높다
- Perplexity는 모델이 각 단계에서 불확실해하는 효과적인 어휘 크기로 이해할 수 있어 더 해석하기 쉬운 것으로 여겨집니다(위 예제에서는 48,725개의 단어 또는 토큰입니다)
- 다시 말해, perplexity는 모델이 예측한 확률 분포가 데이터셋의 실제 단어 분포와 얼마나 잘 일치하는지를 측정합니다
- 손실과 마찬가지로, 낮은 perplexity는 모델 예측이 실제 분포에 더 가깝다는 것을 의미합니다

## 5.3 무작위성 제어를 위한 디코딩 전략
- 위의 `generate_text_simple` 함수를 여러 번 실행하더라도 LLM은 항상 같은 출력을 생성합니다
- 이제 `generate_text_simple`을 수정하는 두 가지 개념, 소위 디코딩 전략인 *temperature scaling*과 *top-k* 샘플링을 소개합니다
- 이들은 모델이 생성된 텍스트의 무작위성과 다양성을 제어할 수 있게 해줍니다
### 5.3.1 Temperature scaling
- 온도 스케일링이라는 개념을 통해 분포와 선택 과정을 제어할 수 있습니다
- "Temperature scaling"은 로짓(logits)을 0보다 큰 수로 나누는 것을 의미하는 멋진 용어일 뿐입니다
- 1보다 큰 온도는 softmax 적용 후 더 균등하게 분포된 토큰 확률을 만듭니다
- 1보다 작은 온도는 softmax 적용 후 더 확신에 찬(더 날카롭거나 더 뾰족한) 분포를 만듭니다
### Temperature scaling
-Temperature scaling은 softmax를 적용하기 전에 로짓(logits)을 나누는 것입니다.
- 운영 체제에 따라 결과 드롭아웃 출력이 다르게 보일 수 있습니다. 이러한 불일치에 대해서는 [PyTorch 이슈 트래커](https://github.com/pytorch/pytorch/issues/121595)에서 더 자세히 읽을 수 있습니다
![Pasted image 20250721110220.png](attachments/Pasted%20image%2020250721110220.png)
- 온도 0.1을 통한 재스케일링이 `torch.argmax`에 가까운 더 날카로운 분포를 만들어서 가장 가능성이 높은 단어가 거의 항상 선택되는 것을 볼 수 있습니다:
- 온도 5를 통한 재스케일링된 확률은 더 균등하게 분포됩니다:
- LLM 입력 "every effort moves you"를 가정하면, 위의 접근법을 사용하여 때때로 "every effort moves you pizza"와 같은 무의미한 텍스트가 나올 수 있습니다. 이는 1000번 중 32번(3.2%)의 빈도로 발생합니다
### 5.3.2 Top-k 샘플링
- 출력 다양성을 높이기 위해 더 높은 온도를 사용할 수 있으면서도 무의미한 문장의 확률을 줄이기 위해, 샘플링된 토큰을 가장 가능성이 높은 상위 k개 토큰으로 제한할 수 있습니다:
![](https://sebastianraschka.com/images/LLMs-from-scratch-images/ch05_compressed/topk.webp)
