# 연산자 오버로딩이란

### 1. 오버로딩 정의

같은 함수 이름을 가지고 있으나 매개변수, return type 등의 특징은 다른 여러개의 함수를 만들 수 있다.

**연산자 오버로딩**

```cpp
// 예를 들어)
char *s = "hello ";
char *c = "world";
s = s + c;
```

C++에서는 이것이 불가능하다. 이것을 가능게 하는 기능이 바로 **연산자 오버로딩**이다. 

### 2. operator 키워드 정의

C++에서 operator를 사용하여 이미 존재하는 대부분의 연산자의 기능을 재정의가 가능하다.

예를 들어)  "+”를 연산자 오버로딩 해보자

1) 연산자 함수 +의 리턴형은 void이다.

2) 함수의 파라미터는 “CComplex c”

→ **void operator+(CComplex c)**;

하고 동일하다. 

```cpp
class CComplex 
{
private:
float a,b;

public:
...
	void operator+(CComplex c)
	{
		a+=c.a;
		b+=c.b;
	}
}

void main()
{
	CComplex a(1.,2.), b(3.,4.);
	a.operator+(b);
}
```

	a.operator+(b); 

위의 연산자 호출은 **명시적 호출**이라고 한다. 하지만 이보다 더 좋은 사용 방법이 있다.

a + b

로 쓸 수 있다. 위의 명시적 호출과 결과가 같다.

### 4. friend 연산자 사용

```cpp
class CComplex
{
public:
	CComplex(float ta = 0, float tb = 0) {
		a = ta, b = tb;
	}
	CComplex operator+(CComplex a, CComplex b);
private:
	float a, b;
};

CComplex operator+(CComplex x, CComplex y)
{
	CComplex t(x.a,x.b)
	
	t.a+= y.a;
	t.b+= y.b;
	return t;
}
```

위 코드 처럼 정의는 불가능하다.!!! 위 operator은 일반 함수이다. 특정 클래스의 멤버 함수가 아니다

x+y → x.operator(y)로 변환되므로, 함축적 호출에서 2개의 파라미터로 명시하는 것은 불가능하다. 만약 위 연산자 함수가 정의된다면, **x.operator(y,z)를 호출**하기 위해서는 “ **x+y,z** “ 이렇게 명시해야 한다. 하지만 이것은 **문법적 에러**이다.

→ friend 키워드를 사용하면 문제가 풀린다.

**일반 함수의 연산자 오버로딩은 반드시 친구 함수로 구현하라**

```cpp
#include <stdio.h>

class CComplex
{
public:
	CComplex(float ta = 0, float tb = 0) {
		a = ta, b = tb;
	}
	void Print();
	friend CComplex operator+(CComplex x, CComplex y);
	friend CComplex operator+(float f, CComplex y);
private:
	float a, b;
};

void CComplex::Print() {
	printf("%2.2f%+%2.2fi\n", a, b);
}

CComplex operator+(CComplex x, CComplex y) {
// 이제 operator+에서 CComplex의 private 멤버를 접근하는 것이 허용된다.
// CComplex는 이 함수를 친구로 생각하고 있는 것이다.
	CComplex t(x.a, x.b);

	t.a += y.a;
	t.b += y.b;

	return t;
}

CComplex operator+(float f, CComplex y) {
	CComplex t(f);

	t.a += y.a;
	t.b += y.b;

	return t;
}

int main() {
	CComplex x(1., 2.), y(3., 4.), z;
	z = 3 + y;
	z = x + y;
	z.Print();
}
```

실수부와 허수부는 private에 놓아야 한다.

### 6. functor 정의

functor 자체는 함수들만을 멤버로 가지는 클래스 입니다.

**알지 못하는 클래스의 객체를 만들지 않고도 만들어질 함수를 미리 호출하는 코드를 작성하는 일을 할 수 있다.**

```cpp
#include <stdio.h>

class Ctest
{
public:
	int value;

	Ctest()
	{
		value = 0;
		printf("constructor\n");
	}
	int operator()(int a, int b)
	{
		printf("compare\n");
		return a > b;
	}

	int operator()(int i)
	{
		printf("assign\n");
		value = i;
		return value;
	}
private:

};

void main()
{
	Ctest a;
	Ctest b;

	b.value = 3;
	b = Ctest();
	printf("%d\n", b.value);
}
```

위 출력 결과는

```cpp
constructor
constructor
constructor
0
```

이다.

```cpp
...
void Test(Ctest a)
{
	a(3,4)
}
void main()
{
	Test(Ctest())
}
```

위 실행 결과는

```cpp
constructor
compare
```

위의 코드는 템플릿을 사용하지 않았다. 하지만 템플릿을 사용할 수 있다. 

형을 파라미터로 받는 것이 바로 템플릿이고, 위의 경우 CTest에 해당하는 것이 STL에서는 functor라고 한다.
