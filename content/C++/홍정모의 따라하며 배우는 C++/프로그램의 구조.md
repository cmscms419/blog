# 프로그램의 구조

### **function(**함수)

- 입력 → 결과 (입력과 출력이 중요하다.)

### int main()

- 오퍼레이팅 시스템이 맨 처음 찾는 함수
- 빈 줄은 컴파일러가 무시
- int 함수명 : 출력값
- {} 중괄호 : 코드를 작성하는 곳
- return 0  : 출력값을 돌려준다는 → 0을 돌려준다.
- “ **;** “ : 문장의 마침

### expresion(표현식)

- 식, 하나 이상의 값으로 표현 될 수 있는 코드
- int x = 2 : 정수형 데이터를 메모리 공간 할당, 2라는 정수 할당
- 2,3,4 : 리터럴
- x, y, z : 변수

### **statement(**명령문)

실행가능한 최소한의 독립된 코드 

명령문이 표현식 보다 더 큰 개념이다.

## 주석 잘 다는 법

주석 : 컴파일러가 무시하게 하는 코드

하나 또는 여러 코드를 주석 처리할 수 있다.

// : 한줄

/**/ : 영역으로 주석처리 한다.

## 변수와의 만남

**객체** : 메모리에 저장 되는 것

**변수** : 객체를 부르기 위한 이름

**변수명** : 변수의 이름

= : assignment (대입, ~ 할당해 준다.) ‘같다’라는 의미가 아니다.

& : ampersand : 변수의 주소값을 알려준다.

컴퓨터는 내부적으로 주소를 사용한다.

사람이 사용할 때는 변수명을 사용한다.

l-value : 왼편, 주소를 가지고 있는 것

r-value : 오른편 : 임시적 주소을 가진다. → 값이 사라진다.

초기화가 안되면, run-time error가 나올 수 도 있다.

int x = 123 → ‘=’ initialization이다.

int x(123) → 이런 초기화도 있다.

초기화는 하는 것이 좋다

## 입출력 스트림과의 첫 만남

#include <iostream> // cout, cin, endl

- **cin**

std::cin >> 변수

>> input operater

- **cout**

std::cout << “123123” << std::endl;

“123123”이 “<<”을 통해서 std::cout에 타고 간다.

- **endl**

endl 줄을 끝냈다.

\t  줄맞춤을 해준다. 한 글자처럼 인식된다.

중괄호 안에 using namespace std해주면, 중괄호 안에는 std를 생략할 수 있다.

변수가 담을 수 있는 최대 크기가 있다.

## 함수

함수는 입력과 출력값이 있어야 한다.

**매개변수**는 입력값

**return**는 출력값

선언 : 함수의 이름, 입력, 출력

→ main 앞에 적으므로, main 함수 뒤에 함수를 정의해도 사용할 수 있다.

## 키워드와 식별자 이름짓기

- main은 함수이름으로 사용하면 안된다.
- 특수문자 금지
- 변수명에 빈칸이 있으면 안됨 → ‘_’을 사용
- 대문자는 변수명으로 짓지는 않는다, 매크로로 사용된다.
- 함수명 : 대문자로 많이 쓴다, 동사 + 명사를 많이쓴다.

## 지역 범위

중괄호의 역할은 영역을 분리해 준다.

중괄호 안의 변수는 중괄호를 나가면, 변수는 사라진다.

스택 메모리에 반납되기 때문이다.

## 연산자와의 첫 만남

리터럴 : 문자열, 숫자, 표현식, 연산의 대상자

조건 연산자는 삼항 연산자이다.

## 선언과 정의의 분리

main 뒤에 함수를 정의하면, main는 그 함수를 알 수 가 없다.

main 앞에 출력과 입력이 무엇인지 알려준다.

int add(int a, int b); → 전방선언

int add(int a, int b)

{

…

} 

→ 정의

선언과 정의를 분리할 수 있다.

선언을 하면, 함수 정의 부분으로 이동 할 수 있기 때문이다.

정의 : 함수의 행동을 결정한다

## 헤더 파일 만들기

<> : 위치에 있는 파일 위치가 다르기 때문이다.

“” :  사용자 지정했기 때문입니다.

클래스는 파일을 쪼개서, 사용하는 것이 일반적이다.

## 헤더 가드

헤더는 있지만, 헤더가 가리키는 몸통이 없어서 링크 에러가 나온다.

#define MY_ADD

#endif

→ 중복을 방지하다.

비주얼 스튜디오는 “#pragma once”을 통해서 중복을 방지합니다.

## 전처리기와의 첫 만남

**Preprocessor**

매크로는 거의 대문자로 쓴다.

#define 문자를 교체해준다.

#ifdef ‘’ ~ #endif → 정의 되어 있으면, 실행

#ifndef ‘’ ~ #endif → 정의 안되어 있으면, 실행

using namespace는 조심히 써야 한다.

define의 영향력은 파일 하나에 영향을 준다.

다른 파일에는 영향을 주지 않는다.

define의 효력범위를 설정해야 한다.
