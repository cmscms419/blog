# Introduction to CUDA Programming

# Program vs Process vs thread

## Program

여러 개의 명령어를 만들어서, 하드디스크에 저장한 다음

프로그램을 실행할 때, 저장한 프로그램을 불러와서 메모리에서 프로그램을 실행한다.

## Process

프로그램을 메모리에 올려서, 프로그램을 진행(process)시킨다.

한 프로그램에 프로세스를 여러 개를 사용하는 경우도 있다.(특수한 경우이다.)

→ TCP 통신 같이 특수한 방법으로 프로세스끼리 데이터를 전달할 수 있다. 하지만 느리다.

프로세스는 서로 독립적인 존재로 통신을 가급적 하지 않는 전제로 구동한다.

서로 간의 직접적으로 데이터를 주고받기가 어렵다.

프로세스를 관리하기 쉽게 하기 위해서 서로 따로 존재한다.

## thread

한 프로세스 안에서 여러 개의 쓰레드가 존재할 수 있다.

쓰레드끼리는 서로 메모리 공간을 같이 사용한다.

→ 쓰레드를 여러 개 사용하는 방식을 Multithreading이라고 한다.

쓰레드의 스케줄을 짜줄 수 있다.
